# CTI AI Trust Gateway 0.1.0b2

Release status: **corrective public research-beta prerelease; not production ready**.

`0.1.0b2` supersedes `0.1.0b1` for new downloads. It makes narrowly scoped maintenance
corrections and does not change the gateway's verification, policy, review, or export behavior.

## Corrective changes

- Corrects the Apache-2.0 license attribution to the project contributors.
- Reconciles release-readiness and historical handoff documentation with the already published
  `v0.1.0b1` prerelease.
- Adds a private-security-reporting contact link and disables unsafe blank public issues.
- Requires pull requests, resolved conversations, an up-to-date branch, and all four mandatory CI
  checks before changes reach `main`; force pushes and branch deletion are blocked.
- Updates package, API, citation, container-test, and release metadata to `0.1.0b2`.

## Verification

The mandatory [GitHub Actions workflow](https://github.com/ag2020sa/cti-ai-trust-gateway/actions/workflows/ci.yml)
covers these exact checks:

- Python 3.12 quality and tests
- Python 3.13 quality and tests
- Build and verify distributions
- Docker build and health

Release artifacts are rebuilt from the final green `main` commit, checked with Twine, installed and
tested from outside the checkout, and published with a SHA-256 checksum manifest. The hosted
release records the exact final workflow run.

## Security and compatibility

The research-beta limitations are unchanged. The application has no authentication, role
authorization, CSRF protection, malware scanner, parser sandbox, rate limiting, encrypted object
storage, or production support commitment. Do not expose it publicly or process real or sensitive
CTI. This release is not a compliance certification. Report suspected vulnerabilities only through
GitHub Private Vulnerability Reporting.

Python `>=3.12,<3.14` is supported. Python 3.13 is locally verified; Python 3.12 and Docker are
verified by the mandatory workflow. No package-index or container-registry publication is part of
this release.

## Upgrade notes

Install `v0.1.0b2` for new evaluations. Runtime SQLite data is excluded from release artifacts;
recreate demo data after installation rather than copying a development database. No data migration
or core behavior change is introduced by this corrective beta.
