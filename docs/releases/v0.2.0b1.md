# CTI AI Trust Gateway v0.2.0b1 — OpenCTI Draft Delivery Research Beta

> **OpenCTI Draft delivery is EXPERIMENTAL, disabled by default, contract-tested, and not live-verified.**

`v0.2.0b1` is an audited research prerelease for local evaluation. It is not production-ready, a
compliance certification, or evidence of live OpenCTI interoperability.

## The problem this gateway addresses

AI-produced threat intelligence can look structurally valid while changing an observable,
inventing attribution, overstating confidence, or creating an unsupported relationship. The
gateway sits between an AI CTI producer and an operational platform. It binds candidate claims to
the supplied source, applies explicit policy, preserves analyst decisions, and exports only the
approved graph.

This release can stage that exact approved graph in an isolated OpenCTI Draft. It never promotes
objects to the live graph and never substitutes its verdict for the human validation that remains
mandatory inside OpenCTI.

## Approved-only Draft delivery

The delivery boundary keeps four decisions separate:

1. `VALID_STIX` — the candidate passed the pinned STIX validator.
2. `OPENCTI_COMPATIBLE` — the graph conforms to the pinned OpenCTI profile.
3. `EVIDENCE_VERIFIED` — selected assertions are grounded in the source or explicitly accepted
   under the review policy.
4. `DELIVERY_AUTHORIZED` — approval closure, integrity, policy, markings, and limits all pass.

The deterministic `ApprovedArtifact` contains only the exact permitted dependency-closed graph and
a bounded provenance report. The deterministic `DeliveryPlan` binds the artifact, approval and
analysis snapshots, compatibility profile, destination fingerprint, options, expiry, markings, and
ordered operations. Execution requires confirmation with the complete stored SHA-256 integrity
digest; the digest is not a signature.

Live delivery is CLI-only and requires triple opt-in:

1. `CTI_GATEWAY_OPENCTI_ENABLED=true`;
2. the `deliver --execute` flag;
3. the full stored plan SHA-256 supplied with `--confirm-plan-sha256`.

The unauthenticated API and UI can inspect readiness, create offline plans, and read history, but
cannot probe, deliver, or reconcile. OpenCTI analyst review and approval of the Draft remain
mandatory. There is no automatic live-graph promotion.

## Failure and security model

Ambiguous or partial remote outcomes become `UNKNOWN` or `PARTIAL`. The gateway does not blindly
retry a submission and makes no exactly-once guarantee. Reconciliation is explicit and status-only;
it never resends or promotes content.

Network controls require verified TLS, exact destination host-and-port allowlisting, direct
connections that do not inherit proxy settings, blocked redirects, IDNA-normalized hosts, and
validation of every DNS answer. Metadata, link-local, multicast, reserved, and unspecified targets
are blocked; private and loopback destinations require explicit local-test opt-in. Tokens stay in
environment-backed secret fields, are excluded from plans and logs, and are redacted from bounded
error messages. Request, response, object, byte, timeout, polling, and trust-store limits fail
closed. Destination platform, connector identity/type/scope/state, markings, and the approved
artifact are revalidated immediately before mutation.

## Install this GitHub prerelease

No package is published to PyPI. Download the exact release artifacts and verify the checksum
manifest before installing:

```bash
curl -fLO https://github.com/ag2020sa/cti-ai-trust-gateway/releases/download/v0.2.0b1/cti_ai_trust_gateway-0.2.0b1-py3-none-any.whl
curl -fLO https://github.com/ag2020sa/cti-ai-trust-gateway/releases/download/v0.2.0b1/cti_ai_trust_gateway-0.2.0b1.tar.gz
curl -fLO https://github.com/ag2020sa/cti-ai-trust-gateway/releases/download/v0.2.0b1/SHA256SUMS.txt
sha256sum --check SHA256SUMS.txt
python -m pip install ./cti_ai_trust_gateway-0.2.0b1-py3-none-any.whl
```

The checksum is an integrity check, not a cryptographic publisher signature.

## Current limitations

- No real or disposable OpenCTI instance was contacted; the pinned public contract is verified
  with deterministic offline doubles only. A future `v0.2.0b2` is reserved for work following an
  isolated live OpenCTI Draft test. No stable release is committed.
- This research beta has no authentication, RBAC, or CSRF protection. Do not expose it publicly.
- The API and UI cannot initiate live delivery.
- Do not use production credentials, real sensitive reports, private indicators, or customer data.
- There is no malware sandbox, parser-process isolation, rate limiting, signed audit store,
  production monitoring, automatic cleanup, atomic cross-system transaction, or exactly-once
  delivery guarantee.
- No package is published to PyPI, and no image is published to a container registry.
- Compatibility pins will age and must be independently revalidated for other OpenCTI versions.
- No copied proprietary Ariane or OpenCTI Enterprise implementation or fixture is included.

**OpenCTI and STIX names are used solely for compatibility identification; no endorsement is implied.**
