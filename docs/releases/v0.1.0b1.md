# CTI AI Trust Gateway 0.1.0b1

Release status: **verified research beta candidate; not yet tagged or published**.

`0.1.0b1` is the first public-facing research MVP candidate for testing whether AI-produced STIX
claims remain faithful to a supplied source document. It is deliberately local-first, conservative,
and unsuitable for production or untrusted-network exposure.

Repository: [github.com/ag2020sa/cti-ai-trust-gateway](https://github.com/ag2020sa/cti-ai-trust-gateway)

## What is included

- TXT, Markdown, and bounded PDF extraction with source hashing, page/offset evidence, language
  detection, and explainable document-security findings.
- Offline STIX 2.1 validation using 57 bundled OASIS schema files pinned to commit
  `c4f8d589acf2bdb3783655c89e0ffb6e150006ae` and aggregate SHA-256
  `43c2bf45bbaeeb44e5852553abffdebeaaa1584111d92d8a8d3a3101d8bd220f`.
- Atomic evidence checks for observables, vulnerabilities, ATT&CK references, entities,
  confidence, and relationships.
- Explainable PASS, REVIEW, REJECT, QUARANTINE, and ABSTAIN verdicts with fail-closed export rules.
- Local API, CLI, analyst UI, evidence manifests, and hash-chained review audit history.
- Deterministic bilingual fixtures, a 100-mutation synthetic benchmark, and an independent
  adversarial suite.
- Source distribution and universal wheel with the UI, policies, provenance, and offline schemas
  packaged as runtime resources.

## Install and verify the candidate artifacts

From the repository root after a clean build:

```bash
python -m pip install "build>=1.2,<2" "twine>=6,<7"
python -m build
python -m twine check dist/*
python -m venv .venv-release
.venv-release/bin/python -m pip install dist/cti_ai_trust_gateway-0.1.0b1-py3-none-any.whl
.venv-release/bin/cti-trust demo
```

On Windows, the virtual-environment executables are under `.venv-release\Scripts\`.

No package has been uploaded to a package index. The supported Python metadata is `>=3.12,<3.14`;
Python 3.13.5 is verified locally and in CI, and Python 3.12 is verified in CI.

## Security posture and limitations

The default path makes no semantic-provider network call, external model access is disabled by
default, and mandatory validation cannot silently degrade into PASS. REJECT and QUARANTINE cases
cannot export candidate objects or be converted to accepted cases through review. Invalid,
unavailable, corrupted, failed, or skipped schema validation also cannot produce PASS.

This beta has no authentication, role authorization, CSRF protection, parser sandbox, malware
scanner, rate limiting, encrypted object storage, signed external audit retention, monitoring, or
production support commitment. Do not expose it publicly and do not process real or sensitive CTI.
See [SECURITY.md](../../SECURITY.md) and [the threat model](../threat-model.md).

## Upgrade notes

This is the first packaged beta, so there is no supported upgrade path from an earlier release.
Runtime SQLite data is intentionally excluded from release artifacts. Recreate demo data after
installation rather than copying development databases into the beta.
